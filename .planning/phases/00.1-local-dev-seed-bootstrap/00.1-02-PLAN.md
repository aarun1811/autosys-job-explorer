    Hardcode the schema-user passwords identically to those in `.env.example` (per Open Question 2 recommendation in RESEARCH.md — env-var substitution does not work inside files passed to `/container-entrypoint-initdb.d/`). Exact file contents:

    ```
    -- Source: https://github.com/gvenzl/oci-oracle-free#startup-script-support
    -- Runs ONCE on first container boot when datafile volume is empty.
    -- Idempotency note: re-running this requires `docker compose down -v` to wipe the named volume.

    ALTER SESSION SET CONTAINER = FREEPDB1;

    CREATE USER rectrace IDENTIFIED BY rectrace_pwd
      DEFAULT TABLESPACE USERS QUOTA UNLIMITED ON USERS;
    GRANT CONNECT, RESOURCE TO rectrace;

    CREATE USER autosys IDENTIFIED BY autosys_pwd
      DEFAULT TABLESPACE USERS QUOTA UNLIMITED ON USERS;
    GRANT CONNECT, RESOURCE TO autosys;

    CREATE USER reconmgmt IDENTIFIED BY reconmgmt_pwd
      DEFAULT TABLESPACE USERS QUOTA UNLIMITED ON USERS;
    GRANT CONNECT, RESOURCE TO reconmgmt;

    CREATE USER recportal IDENTIFIED BY recportal_pwd
      DEFAULT TABLESPACE USERS QUOTA UNLIMITED ON USERS;
    GRANT CONNECT, RESOURCE TO recportal;
    ```

    Note: schema names are LOWERCASE because Oracle's case-insensitive identifier resolution stores them uppercase regardless (`RECTRACE`, `AUTOSYS`, etc. in `dba_users`). The Java code in `TlmStatsV2Service.java` uses `reconmgmt.mr_csum_*` lowercase-qualified — both work because the resolved name is uppercase.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| host filesystem → Oracle container | Files in `./init/` are mounted into the container and executed by Oracle SYS during first boot. Untrusted contents could subvert the DB. |
| docker network → host localhost ports | Anyone with local network access to ports 1521/9200 can connect (no auth on ES; default password on Oracle). Acceptable for laptop-only dev. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-00.1-04 | Tampering / Elevation of Privilege | Bind-mounted Oracle init script | mitigate | Mount the init script read-only (`:ro` suffix in the volume entry). Acceptance criteria assert `grep -q ':ro$'` on the init mount line. Init script content is authored in this repo and reviewed via PR. |
| T-00.1-05 | Information Disclosure | Oracle SYSTEM password committed in `docker-compose.yml` (`ORACLE_PASSWORD: oracle`) | accept | Local-dev only; the dev-only password `oracle` matches what Phase 1's `application-local.properties` will use. Sibling repo is never deployed; if pushed to a personal remote, the value is non-secret per D-0.1.3. |
| T-00.1-06 | Denial of Service | ES OOM / Apple Silicon SIGILL | mitigate | `-Xms512m -Xmx512m` caps heap; `-XX:UseSVE=0` prevents the known Apple Silicon SIGILL crash (RESEARCH.md Pitfall 1). |
</threat_model>
