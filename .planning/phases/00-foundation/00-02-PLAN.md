## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| test profile → production config | Test profile must not activate in production; would suppress DatabaseConfig bean, causing missing DataSource at runtime |
| application-test.properties → production classpath | File is in src/test/resources/ — Spring Boot Maven plugin excludes test-scope resources from the production JAR |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-00-04 | Elevation of Privilege | DatabaseConfig @Profile("!test") | accept | Profile guard is `"!test"` — activates in all profiles except `test`. Production, UAT, dev profiles all activate the real DatabaseConfig. Low risk. |
| T-00-05 | Information Disclosure | application-test.properties placeholder credentials | accept | Placeholder values (`jdbc:oracle:thin:@test-placeholder`, username=`test`) are non-functional test tokens, not real credentials. File never packaged in production JAR (src/test/resources/). |
| T-00-06 | Denial of Service | maven.test.skip removal in tlm-stats module | accept | Same as T-00-03 — a broken test fails the build. This is the intended gate. |
</threat_model>
