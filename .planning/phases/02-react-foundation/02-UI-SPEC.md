## Spacing Scale

Declared values — all multiples of 4. Source: Tailwind v4 defaults + shadcn `@theme inline` radius scale.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline element spacing (grid cell icon padding) |
| sm | 8px | Button internal padding, compact list item gaps |
| md | 16px | Default card/section padding, form field gaps |
| lg | 24px | Header/footer padding, section separators |
| xl | 32px | Layout column gaps |
| 2xl | 48px | Major section breaks (header-to-content gap) |
| 3xl | 64px | Page-level vertical rhythm |

**Exceptions:**

- AG-Grid row height: 36px (`--ag-row-height: 36px`) — not a spacing scale multiple; dictated by recviz AG-Grid token bridge (RESEARCH.md Code Examples).
- AG-Grid header height: 40px (`--ag-header-height: 40px`) — same rationale.
- AG-Grid cell horizontal padding: 12px (`--ag-cell-horizontal-padding: 12px`) — same rationale.
- Theme toggle button: 32px icon-sm size (shadcn `size="icon-sm"` Button) — dictated by recviz header pattern.
- Touch target minimum: 32px (icon-sm) — internal desktop-only app; no mobile; 32px meets desktop usability.

## Typography

Source: recviz AG-Grid token bridge (RESEARCH.md) + shadcn new-york defaults. Phase 2 does not introduce
a custom font stack; it inherits Inter from the AG-Grid bridge declaration.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px | 400 (regular) | 1.5 | Default prose, form labels, grid cell text, toast body |
| Label / Caption | 12px | 500 (medium) | 1.4 | AG-Grid header text (`--ag-header-font-size: 12px; --ag-header-font-weight: 500`), metadata labels |

### 60/30/10 Distribution

| Role | Distribution | Elements |
|------|-------------|---------|
| Dominant surface (60%) | `var(--background)` | Page body, grid rows, card backgrounds, header background |
| Secondary surface (30%) | `var(--secondary)`, `var(--muted)`, `var(--sidebar)` | AG-Grid header row, navigation, sidebar, secondary buttons |

**Accent reserved for (explicit list):**
1. Theme toggle button active state

Accent is NOT used for: body text, borders, backgrounds, decorative elements.

- **Layout:** Single row, `border-t`, `px-4 py-2`, `text-muted-foreground text-[12px]`.

## Component Specs

### ThemeSwitch

- **Trigger:** `Button size="icon-sm" variant="ghost"` — matches recviz `theme-switch.tsx`.
- **Icon:** `MoonIcon` when resolved theme is `light`; `SunIcon` when `dark`. Both from `lucide-react`.
- **Screen reader:** `<span className="sr-only">Toggle theme</span>` always present.
- **Mount guard:** Renders `null` until `mounted` is `true` (prevents hydration flash — mirrors recviz pattern exactly).
- **State owner:** Custom `ThemeProvider` React context (NOT `next-themes` ThemeProvider, even though `next-themes` is installed). `useTheme()` hook returns `{ resolvedTheme, setTheme }`.
- **Theme provider:** `src/components/layout/theme-provider.tsx` — copy recviz pattern verbatim with `STORAGE_KEY = 'rectrace-theme'` (not `'recviz-theme'`).
- **Theme effect:** `document.documentElement.classList.remove('light', 'dark'); classList.add(resolvedTheme)` — must produce `.dark` class for `@custom-variant dark (&:is(.dark *))` to engage.

### SmokeGrid (AG-Grid SSRM)

- **Theme class:** `ag-theme-quartz` on the container div.
- **Height:** `height: calc(100vh - var(--header-height) - <footer-height>)` — fill remaining viewport. Use `h-[calc(100vh-6rem)]` as Tailwind approximation; executor adjusts once heights are measured.
- **rowModelType:** `"serverSide"` — mandatory for SSRM.
- **Modules registered:** `[ServerSideRowModelModule]` — only this module in Phase 2. License must be set before registration.
- **License:** `LicenseManager.setLicenseKey(import.meta.env.VITE_AG_GRID_LICENSE_KEY ?? '')` as first executable line in `main.tsx`.
- **Columns (Phase 2 smoke — minimal, not final):**

  | Field | Header | Width |
  |-------|--------|-------|
  | `jobName` | Job Name | 200px |
  | `fileName` | File Name | 200px |
  | `machine` | Machine | 150px |
  | `status` | Status | 120px |

  Phase 3 ports real column definitions from Angular `search-config-v4.json`.

- **Empty state:** Rendered as AG-Grid overlay (see Copywriting Contract below). Phase 2 uses AG-Grid's built-in `overlayNoRowsTemplate`.
- **Loading state:** AG-Grid built-in loading overlay (no custom spinner in Phase 2).
- **Error state:** `params.fail()` called on fetch error; Sonner toast fires via `QueryCache.onError` with correlation ID (see Copywriting Contract).
- **Row height:** 36px (set via `--ag-row-height` CSS token, not `rowHeight` prop — let the token drive it).
- **Header height:** 40px (set via `--ag-header-height` CSS token).

### Error Display (Correlation ID)

**Shape:** Sonner toast, NOT an inline alert, NOT an error boundary screen, for Phase 2.

Rationale: Phase 2 only has one data surface (the smoke grid). A non-blocking toast is appropriate. Phase 3 will inherit this same toast pattern for search errors (SEARCH-06 anchor). If a full-page error boundary is needed (e.g., catastrophic app init failure), use React's default ErrorBoundary — but the correlation ID pattern lives in the toast.

- **Toast library:** `sonner` (already vendored).
- **Trigger:** `QueryCache.onError` in `queryClient.ts` fires `toast.error(...)`.
- **Position:** `"bottom-right"` (matches recviz `Toaster` props).
- **Dismissed by:** User click or 4s auto-dismiss.
- **Toaster mount:** In `__root.tsx`: `<Toaster richColors position="bottom-right" />`.

## Interaction Contract

### Dark / Light Toggle

- **Default:** System preference (`prefers-color-scheme`) on first load.
- **Persistence:** `localStorage` key `rectrace-theme` (string: `"light"` | `"dark"` | `"system"`).
- **Toggle behavior:** Single click cycles between `light` and `dark` only (not three-state in Phase 2 — system is only the initial default, not a selectable toggle state). Matches recviz `ThemeSwitch` which toggles between resolved light and dark.
- **Visual feedback:** Icon swaps immediately (no animation in Phase 2; Phase 8 DESIGN-02 may add transition).
- **Class application:** `.dark` class on `document.documentElement`. Affects all shadcn tokens and AG-Grid bridge tokens simultaneously.
- **Feature parity target (REACT-05):** Equals the Angular app's toggle (dark/light, `localStorage`-persisted, system default).

### Correlation ID Flow

1. `apiFetch(path, init)` in `queryClient.ts` generates `crypto.randomUUID().replace(/-/g, '')` per call.
2. Header `X-Correlation-Id: <32-hex>` sent with every request.
3. Backend Brave propagator reads `X-Correlation-Id` via baggage (`management.tracing.baggage.remote-fields=x-correlation-id`); logs contain `[corrId=<32-hex>]`.
4. On `!res.ok`: `Object.assign(new Error(...), { correlationId })` thrown.
5. `QueryCache.onError` catches, fires `toast.error('Request failed', { description: 'Error reference: <corrId>' })`.
6. User reads the 32-char hex in the toast and quotes it verbatim in a bug report.

### AG-Grid SSRM Datasource Lifecycle

- `getRows` is called by AG-Grid on initial mount and on any sort/filter change.
- `params.success({ rowData, rowCount })` on HTTP 200.
- `params.fail()` on any error (after toast has fired).
- No retry in Phase 2 — `queryClient.defaultOptions.queries.retry: 1` handles query retries; the raw SSRM datasource does not.
- `rowCount` is set to `data.lastRow` (backend field from `SSRMResponseV4.lastRow`). If `-1`, AG-Grid uses infinite scroll.

## ESLint Enforcement Contract

The hex-rejection rule (D-2.8) means:

- `color: '#1a2b3c'` in a TSX file = ESLint error.
- `style={{ color: '#fff' }}` in a TSX file = ESLint error.
- `var(--color-primary)` = allowed.
- `oklch(0.488 0.243 264.376)` = allowed (not a hex literal).
- `components/ui/*.tsx` (vendored shadcn) = exempt via `globalIgnores(['src/components/ui/**'])`.

Selectors and class strings like `"bg-primary"` are Tailwind utilities and are allowed — they are string literals, not hex literals.
