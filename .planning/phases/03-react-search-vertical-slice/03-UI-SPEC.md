## Ported Category

**Selected category: `fileName` (File Name)**

Rationale:
1. `fileName` was the Phase 2 SSRM smoke target — backend wiring is already live-verified end-to-end against this category's endpoint (`/rectrace/api/v4/search/ssrm/fileName`).
2. It has the richest renderer set of all V4 categories: `appIDCellRenderer`, `supportEmailCellRenderer`, `executionOrderButtonRenderer` — three distinct interaction patterns in a single tab.
3. The Phase 0.1 local seed contains `fileName`-keyed rows; SSRM returns verified data immediately.
4. SC#2 requires porting at least one custom renderer — `fileName` satisfies this with three renderer types, giving the executor choices without ambiguity.

**Renderer selected for SC#2 port: `executionOrderButtonRenderer`**

Rationale: It is the most visually distinctive and behaviorally meaningful renderer (button with loading state + modal launch). Porting it proves the custom-renderer registration pipeline, async API call, and Sonner error surface for a cell-level action. The `appIDCellRenderer` (external link) and `supportEmailCellRenderer` (mailto link) are simpler and will be trivially portable by inspection once the harder renderer pattern is proven.

**Parity matrix update (this phase locks these rows):**

| Row | Target | Renderer target |
|-----|--------|----------------|
| File Name search tab | port | in this phase |
| `executionOrderButtonRenderer` | port | in this phase |
| `appIDCellRenderer` | port | in this phase (simpler: external link anchor) |
| `supportEmailCellRenderer` | port | in this phase (simpler: mailto anchor) |

The Angular source uses raw hex `#1a73e8` (google-blue) for the button label and hover states. In React, map to:

- Label text color: `text-primary` (maps to `var(--primary)` = mist OKLCH `0.488 0.243 264.376` in light; `0.424 0.199 265.638` in dark)
- Hover background: `bg-accent` (maps to `var(--accent)`)
- Disabled state: `opacity-50 cursor-not-allowed text-muted-foreground`

This achieves visual parity while keeping the component hex-free.

### Recent Searches Popover

**Trigger:** Input receives focus AND (`input is empty` OR `localStorage has recent searches`).

**Dismiss:** Input loses focus (with a 150ms delay to allow click on popover items) OR user presses Escape OR user starts typing (close popover, show live suggestions instead).

**Storage:** `localStorage` key `rectrace-recent-searches`. Value: JSON array of strings. Max length: 10. On search submission, prepend the new term and slice to 10. Deduplicate: if term already exists, remove existing and prepend. Case-sensitive (match Angular behavior for enterprise terms like job names).

**Popover anatomy (using shadcn `Command` inside `Popover`):**

│ ─────────────────────────────────────────── │  ← Separator

## URL-State Encoding (SEARCH-03)

**Scheme:** Standard query params — no hash, no base64. Chosen for debuggability and human-readability (engineers read support tickets with URLs pasted in; base64 obscures the state).

**Parameters:**

| Param | Value | Example |
|-------|-------|---------|
| `q` | Search term (URL-encoded) | `?q=LOAD-ABC-123` |
| `cat` | Category key | `?cat=fileName` (hardcoded in Phase 3; extensible for Phase 4+) |

**Minimal — only what's needed for deep-link restore:**

For Phase 3, only `q` and `cat` are URL-synced. Column sort order, filter state, and expanded group keys are intentionally **not** URL-synced in Phase 3 — they are session-ephemeral (grid manages them internally). URL sync of deeper grid state is deferred to Phase 4+ design polish.

**Implementation:** TanStack Router `useSearch()` hook with Zod schema validation:

```typescript
const searchSchema = z.object({
  q: z.string().optional(),
  cat: z.string().optional().default('fileName'),
})
```

**On URL change (navigation or paste):** `useEffect` reads `q` from search params; if non-empty, sets input value and immediately fires search. Grid datasource is created with the restored `q` value.

**On search submission:** `router.navigate({ search: { q: term, cat: 'fileName' } })` — replaces current history entry (no back-stack spam on each keystroke; update only on submit, not on type).

**On clear:** Navigate to `{ search: {} }` to remove params.

**Parity with Angular:** Angular uses `this.router.navigate([], { queryParams: { q: searchTerm } })` — same mechanism, same semantics.

## Interaction Contract

### Search Flow

```
User types → [300ms debounce] → (no API call on type in Phase 3)
User clicks "Search" button or presses Enter
  → Set URL param: ?q=term&cat=fileName
  → Call POST /rectrace/api/v4/search/initial with { searchTerm, categoryKey }
  → On success: store response as initialFilter for SSRM datasource
  → Create/refresh SSRM datasource with initialFilter
  → Show grid; hide pre-search empty state
  → Store term in localStorage recent-searches (prepend, dedupe, limit 10)
  → Show result count in toolbar badge (from SSRM lastRow)
```

**Two-step search (matches Angular pattern):**
1. `POST /rectrace/api/v4/search/initial` — returns pre-filter data from Elasticsearch.
2. AG-Grid SSRM calls `POST /rectrace/api/v4/search/ssrm/fileName` with `initialFilter` embedded in the request body.

**Search state machine:**

| State | What user sees |
|-------|----------------|
| `idle` (no search yet) | Pre-search empty state (no grid, no toolbar, no tab bar) |
| `loading` (initial fetch in flight) | Skeleton loader in grid area — 3 rows × full width |
| `success` | Grid with results; toolbar with count; category tab bar |
| `empty` (search returned 0 rows) | Grid area replaced with empty-state card |
| `error` | Sonner toast bottom-right with correlation ID; grid area shows error state card |

### Loading State

**During initial fetch (`POST /initial`):** Show skeleton loader in the grid area. Use 3 `Skeleton` rows at grid row height (36px) across the full grid width, matching the column layout approximately.

Do NOT use a full-page spinner. The skeleton preserves layout continuity and is consistent with recviz's loading pattern for data-dense tables.

**During SSRM row fetch (AG-Grid loading overlay):** AG-Grid's built-in loading overlay fires automatically on every `getRows` call. No custom spinner needed.

**During export:** Export button icon → `Loader2Icon animate-spin`; button `disabled`.

### Error Surfaces

| Error type | Surface |
|-----------|---------|
| Initial fetch (`/initial`) failure | Sonner toast `"Request failed" / "Error reference: {corrId}"` + error state card in grid area (with retry button) |
| SSRM row fetch failure | Sonner toast (same shape) + AG-Grid `params.fail()` (grid shows AG-Grid's built-in error state) |
| ExecutionOrder button fetch failure | Sonner toast (same shape) — no inline error; button re-enables |
| Excel export failure | Sonner toast (same shape) |

**Retry affordance:** Only on initial fetch failure — show a "Retry" `Button size="sm" variant="outline"` in the error state card. On SSRM failures, user can re-run search from the input.

**Correlation ID format in all toasts:** `"Error reference: {32-char lowercase hex}"` — per Phase 2 D-2.9/D-2.11 contract. Uses the `reportRequestFailure(err)` function from `queryClient.ts` (already exported from Phase 2).

### Navigation / State Restoration

**When user pastes a URL with `?q=LOAD-ABC-123&cat=fileName`:**
1. Page loads; TanStack Router reads search params.
2. Search input is pre-populated with "LOAD-ABC-123".
3. Search fires automatically (as Angular does in `initializeQueryParamsSubscription`).
4. Grid shows results.

**When user navigates away (e.g. back button) and returns:**
- TanStack Router re-reads URL params and triggers search again.
- Recent searches in localStorage are preserved.
- Column state (sort/filter) is NOT restored — session-ephemeral.

**When input is cleared:**
- URL params removed.
- Grid unmounted (pre-search empty state shown).
- Recent searches popover available on focus.
