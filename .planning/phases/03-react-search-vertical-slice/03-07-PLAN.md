<objective>
Wire the end-to-end vertical slice. SearchPage orchestrates the search flow:

1. Reads `q` + `cat` from the URL (TanStack Router `useSearch`, Zod-validated by the route).
2. On submit (Enter, Search button click, or recent-search item click): `navigate({ q, cat }, replace: true)` → `apiFetch('/rectrace/api/v4/search/initial?keyword=...')` (GET — Pitfall 4) → store response as `initialFilter` → SearchGrid mounts (key-remounts on subsequent searches per D-3.5).
3. On URL paste with `?q=...`: same handler fires via `useEffect([q, cat])` — deep-link restore (SEARCH-03).
4. Holds `gridApiRef` (via `onGridReady`); SearchToolbar's `onExport` closure calls `gridApi.exportDataAsExcel(...)` (SEARCH-04, D-3.10).
5. On successful submit, pushes the term to `useRecentSearches` (SEARCH-05).
6. All fetch errors route through `reportRequestFailure` (Sonner with correlation ID — SEARCH-06).
7. Error-state card with Try-again button surfaces inside the grid slot when `/initial` fails (UI-SPEC §Error Surfaces).
8. Deletes `SmokeGrid.tsx` + `SmokeGrid.test.tsx` in the same wave that wires SearchGrid in (Pitfall 8 — only after SearchGrid is real).
9. Includes a manual `checkpoint:human-verify` task after automation completes — visual UAT for the live end-to-end flow against the local seed (the same 5/5 PASS pattern Phase 2 used to close, per STATE.md Performance Metrics).

Purpose: This is the wave where every Phase 3 requirement crosses the finish line as a working user experience. SEARCH-07 (base path `/rectrace/`) inherits Phase 2 D-2.4 — no code change here, just verification at the checkpoint.

Output: 3 component/route files (search.tsx route, index.tsx redirect, SearchPage.tsx) + 1 test file + 2 deletions + regenerated routeTree.gen.ts + 1 manual UAT checkpoint.

|----------|-------------|
| URL → React | Zod validateSearch on the route parses { q?, cat? }; downstream code reads typed values only |
| User submit → backend (/initial GET) | encodeURIComponent on keyword prevents URL injection; backend is the second validation gate |
| Backend response → React (initialFilter) | InitialSearchResponseV4Schema.parse() rejects malformed payloads at the trust boundary; error routes through reportRequestFailure |
| Backend response → AG-Grid cells | Plan 03 renderers (text content via React) handle untrusted strings safely; no innerHTML |
| User → Excel export | buildExportFilename sanitizes term (alphanumeric + underscore + YYYYMMDD) — no path traversal in the local download dialog |
