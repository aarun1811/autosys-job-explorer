## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking] Index create inline before _bulk POST**

- **Found during:** Task 2 — plan 00.1-04 deleted the index after live-validating the mapping (so plan 00.1-06's apply.py could own idempotent index creation). Plan 00.1-05 then calls `es.delete_by_query(index='rectrace_core_index', ...)` which would fail on a missing index.
- **Fix:** Added a conditional `if not es.indices.exists(index='rectrace_core_index'): es.indices.create(index='rectrace_core_index', body=mapping)` block before the delete_by_query / bulk. The mapping is loaded from `es/rectrace_core_index.mapping.json`, so the index is born with the explicit `.keyword` multi-fields at `ignore_above:8192` — Phase 8 hyphen-bug fix prerequisite stays intact.
- **Files modified:** none on disk (one-shot inline Python; the conditional becomes apply.py's responsibility in plan 00.1-06)
- **Rationale:** The plan_text's inline Python assumed the index already existed (plan 00.1-04 originally left it created; plan 00.1-04 changed to delete-after-validate without back-propagating to plan 00.1-05). Inline create is the minimum fix that preserves both the Phase 8 hyphen-bug coverage and plan 00.1-06's idempotency contract.
