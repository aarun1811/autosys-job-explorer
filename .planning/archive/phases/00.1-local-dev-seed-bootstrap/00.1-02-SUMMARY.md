## Decisions Made

- **`container_name` pinning** (BLOCKER-4 fix): rather than parse `docker compose ps --format json` (whose shape varies across Compose versions), pin the names so `docker inspect --format='{{.State.Health.Status}}' rectrace-oracle` is stable across machines and Compose releases.
- **Image-bundled `healthcheck.sh` for Oracle**: gvenzl maintains a probe that correctly waits for SYSTEM availability AND PDB readiness. Hand-rolled `sqlplus`-or-`nc` probes are well-known to give false positives during init-script execution; we trust the image's own probe.
- **`:ro` on the init-script bind mount** (Threat T-00.1-04): the SQL file is authored in this repo, reviewed via PR, and the container only ever reads it — never writes back.
- **Apple-Silicon `-XX:UseSVE=0`** (RESEARCH.md Pitfall 1 / Threat T-00.1-06): the ES JVM SIGILL-crashes on M-series CPUs without this flag; harmless on amd64. Combined with `-Xms512m -Xmx512m` to cap heap.
- **All-lowercase user names in DDL**: Oracle's case-insensitive resolution stores them as `RECTRACE`, `AUTOSYS`, etc. in `dba_users`. The Java code uses `reconmgmt.mr_csum_*` (lowercase-qualified) which still resolves correctly. No quoted identifiers, no case-sensitivity surprises.
