## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] `apply_sql_file` stripped trailing `;` from every chunk, breaking PL/SQL blocks**

- **Found during:** Task 2 cycle 1 `--reset`
- **Issue:** The initial `apply_sql_file` helper stripped a trailing `;` from every chunk it executed. That broke `BEGIN ... END;` PL/SQL blocks because oracledb thin-mode `cursor.execute()` requires the `;` after `END` (verified empirically: `BEGIN NULL; END` raises ORA-06550 / PLS-00103, while `BEGIN NULL; END;` succeeds). The first `--reset` failed immediately on `schema/01-rectrace.sql`'s first DROP-CASCADE block.
- **Fix:** Removed the unconditional `;`-strip; kept only the rare end-of-file `\n/` (no trailing newline) normalization. Plain DDL accepts a trailing `;` without complaint, so the simpler-and-correct rule is to leave chunk endings untouched.
- **Files modified:** `/Users/aarun/Workspace/Projects/rectrace-local-dev/apply.py`
- **Commit:** `54c5882`

No Rule-2/3 issues, no Rule-4 architectural escalations, no checkpoints. The plan's split-task structure (Task 1 structural / Task 2 live) made this bug cleanly bisectable as designed under WARNING-2.
