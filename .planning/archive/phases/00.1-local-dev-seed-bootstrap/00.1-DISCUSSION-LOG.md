## Verification of user's Oracle Instant Client claim

User initially said: "i have oracle instant client installed i think."

Initial broad-path scan (`/opt/oracle`, `/usr/local/lib/instantclient*`, `$ORACLE_HOME`, `mdfind -name sqlplus`) returned nothing — preliminary CONTEXT.md draft said Instant Client was NOT installed.

User pushed back with a hint: "check /Users/aarun/oracle and /Users/oracle/oradiag_aarun path bro. you'll find oracle instant client i think."

Re-verified at `/Users/aarun/oracle/instantclient/`:
- Found: Instant Client **Basic** package version 23.x (`libclntsh.dylib.23.1` symlink present, plus older shimmed versions back to 10.1).
- Not found: `sqlplus` binary (would require the separate SQL*Plus add-on download).
- The `/Users/oracle/oradiag_aarun` path does not exist; that's the diag output dir convention Oracle creates at runtime when applications use the client — appears only if the client has been used.

CONTEXT.md D-0.1.20 and the Specifics entry were corrected. Conclusion: thin mode is still the chosen path because it has no native-library dependency; thick mode via the installed Basic library is an optional README footnote for users who later want thick-mode features.
