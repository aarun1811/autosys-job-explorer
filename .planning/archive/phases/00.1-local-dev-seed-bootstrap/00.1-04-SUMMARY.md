## Why ignore_above: 8192 (and not the default 256)

Per RESEARCH.md Pitfall 3: ES dynamic mapping creates `.keyword` subfields with `ignore_above: 256`. Any value longer than 256 chars on the source field gets silently dropped from the keyword subfield (still indexed as `text`, but the `term`/`match_phrase` on `.keyword` returns zero hits — exactly the user-visible hyphen-bug shape Phase 8 fixes).

8192 is comfortably above any realistic identifier length we'd see in this dataset (production hyphenated identifiers like `SET-XYZ-OPS-123-EU` typically sit at ~32 chars; 8192 is several orders of magnitude over the ceiling) while still being well under Lucene's 32766-byte per-term hard limit.
