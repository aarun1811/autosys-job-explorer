    REACT_START_CMD="npm run dev"

  <action>Per D-8.4 + D-8.5: rewrite `ops/rectrace-ops.sh` end-to-end. Top of file:

  ```bash
  #!/usr/bin/env bash
  # ops/rectrace-ops.sh v2 — runtime ops driven by ops/components.sh registry.
  # Phase 8 OPS-01..03: shellcheck-clean, bash 3.2 + 4/5 portable, readiness-probed start.
  set -euo pipefail

  SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
  REPO_ROOT="$(cd "$SCRIPT_DIR/.." && pwd)"
  # shellcheck source=ops/components.sh
  source "$SCRIPT_DIR/components.sh"

  RUN_DIR="$REPO_ROOT/run"
  LOG_DIR="$REPO_ROOT/logs"
  mkdir -p "$RUN_DIR" "$LOG_DIR"

  READY_TIMEOUT="${RECTRACE_READY_TIMEOUT:-30}"
  READY_INTERVAL="${RECTRACE_READY_INTERVAL:-2}"
  ```

  Then implement helpers (all generic over an `entry` string from `REGISTRY`):

  - `parse_entry <entry>` — splits a `|`-delimited entry into the seven globals `C_NAME`, `C_PORT`, `C_PID`, `C_LOG`, `C_READY`, `C_DIR`, `C_CMD`. Use `IFS='|' read -r C_NAME C_PORT C_PID C_LOG C_READY C_DIR C_CMD <<< "$entry"`.
  - `wait_ready <url> <label>` — polls with timeout `$READY_TIMEOUT`, interval `$READY_INTERVAL`. Uses `curl -fsS -o /dev/null -w '%{http_code}' "$url" 2>/dev/null` and grep for `^2..`. Returns 0 on success, non-zero on timeout. Echo to stderr on failure.
  - `start_one <entry>` — parse, check existing pid (kill -0); if running, echo `already running (pid N)` and return 0; otherwise cd into `$C_DIR` if non-`-`, spawn `$C_CMD` with stdout+stderr to `$C_LOG`, save `$!` to `$C_PID`, return 0. Run-readiness probe AFTER spawn in `do_start`, not inside `start_one`, so `all` can spawn three then probe three.
  - `stop_one <entry>` — `kill` on the stored PID, wait up to 30s for exit, SIGKILL fallback, remove pid file.
  - `status_one <entry>` — print `name: up (pid N, port P)` or `name: down`.
  - `logs_one <entry>` — `tail -f` on `$C_LOG`; explicit error if `logs all` is requested (no multi-tail).

  Dispatcher:

  - Parse `cmd="${1:-}"` and `arg="${2:-all}"`. Validate `cmd` in `{start, stop, restart, status, logs}`; print usage on miss.
  - If `arg == all` → iterate every entry in `REGISTRY` and invoke the verb's `_one` helper.
  - Else → `registry_lookup "$arg"` to get the entry string; error out if missing with `Unknown component: $arg. Valid: $(registry_names | tr '\n' ' ')`.
  - For `start all`: spawn all three first, then probe all three so wall-clock is bounded by the slowest, not summed.
  - For `restart`: `stop_one` then `start_one` then `wait_ready`. For `restart all`: stop all → start all → probe all.
  - `logs` never accepts `all` — explicit error per current script.

  Constraints to verify by shellcheck:
  - Quote every variable expansion.
  - No `cd` without `|| exit 1`.
  - No unquoted `$cmd` when it contains spaces (the `start_cmd` field embeds whitespace — invoke via `eval` ONCE inside `start_one`, with a `# shellcheck disable=SC2086` comment on that exact line and a one-line justification: the field is dev-controlled, sourced from a version-controlled registry, never user input).
  - All array iteration uses `for entry in "${REGISTRY[@]}"; do ... done` (bash 3.2-safe).
  - PID files use `printf '%s\n' "$pid" > "$pid_file"` not `echo -n`.

  Behavior parity: the same `start`/`stop`/`status`/`restart`/`logs` verbs must work for the same three components as the v1 script. The `restart all` and `start all` paths must continue to spawn backend, tlm-stats, react in REGISTRY order (backend first — matches the v1 ordering required by smoke scripts that hit backend immediately).</action>
