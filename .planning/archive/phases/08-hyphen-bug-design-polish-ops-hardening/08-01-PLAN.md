<objective>
Close the daily hyphen-search complaint. Diagnose the existing failure, ship the locked `.keyword`-subfield fix (D-8.2), and lock the regression in JUnit + a live-stack smoke.

Purpose: Hyphenated identifiers (e.g., `RECON-XYZ-42`, `LOAD-ABC-123`, `SET-ABC-123`) are the dominant ID shape in the seed data and in production Autosys job naming. The current wildcard query path returns zero hits for these inputs because `pattern = "*" + keyword.toLowerCase() + "*"` is run against both the text (standard-analyzer-tokenized) field — where atomic tokens cannot match a `*abc-123*` wildcard literal containing a hyphen — and the `.keyword` field, which preserves the original case (`ABC-123`), so the lowercased pattern `*abc-123*` cannot match it either.

Output: One mapping audit, one Java code edit (case-insensitive flag on the `.keyword`-subfield wildcard branch), one regression test, one smoke script. Mapping is **additive only** per D-8.2; no reindex.
</objective>
