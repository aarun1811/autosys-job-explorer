LockingTaskExecutor.executeWithLock signature (research Pattern 2):
```
TaskResult<R> executeWithLock(TaskWithResult<R> task, LockConfiguration config) throws Throwable;
```
- `LockConfiguration(Instant now, String name, Duration lockAtMostFor, Duration lockAtLeastFor)`.
- `TaskResult.wasExecuted()` → boolean; true iff the lock was acquired and the task ran.

BulkIngester construction (research Pattern 3):
```
BulkIngester.of(b -> b
    .client(esClient)
    .maxOperations(def.getBatch().getRows())
    .maxSize(def.getBatch().getBytes())
    .flushInterval(def.getBatch().getFlushMs(), TimeUnit.MILLISECONDS)
    .maxConcurrentRequests(1)
    .listener(new LoaderBulkListener(def.getKey())));
```
- Generic param `<Context>` is the per-op tag passed to the listener — use `String` (e.g. `"jobKey:docId"`) for log lines.
- `BulkIngester.close()` blocks until queue drains; `flush()` forces immediate drain without closing (Pitfall L3 mitigation).
- `BulkIngester.add(BulkOperation op, Context ctx)` is the streaming add.

loaderJdbcTemplate.query(con -> {
    PreparedStatement ps = con.prepareStatement(query);
    ps.setFetchSize(1000);

     - Initial lastFireTime = `Instant.now()` so the very next tick is the cron's first eligible fire from "now", not from epoch.
     - Populate the four maps.
  2. log.info final count of registered jobs.

- `public List<LoaderJobDefV4> dueAt(Instant now)`:
  - For each job, compute `CronExpression.next(lastFireTimes.get(key))`; if that next-fire `<= now`, the job is due.
  - Return list of due jobs in declaration order.
  - Side effect: do NOT update `lastFireTimes` here (that's the ticker's responsibility AFTER acquiring the lock).

- `public BulkIngester<String> ingesterFor(String jobKey)` — direct map lookup; returns null if unknown (caller-checked).

- `public LoaderBulkListener listenerFor(String jobKey)` — direct lookup.

- `public void markFired(String jobKey, Instant when)` — sets `lastFireTimes.put(jobKey, when)`. Called by ticker after acquiring lock.

- `@PreDestroy public void shutdown()`:
  1. log.info "Loader shutting down — flushing N ingesters" (where N = ingesters.size()).
  2. For each entry in ingesters: wrap in `try { ingester.flush(); ingester.close(); } catch (Throwable t) { log.error("flush/close failed for {}", key, t); }` — Pitfall L3 says wrap to prevent unfinished close from crashing shutdown.
  3. log.info "Loader shutdown complete".

Sub-action A — `OracleToEsLoaderJob.java` under `loader/`. Class declaration:
```
@Service
@Profile("!test")
@Slf4j
@RequiredArgsConstructor
public class OracleToEsLoaderJob
```
Fields:
- `private final LoaderJobRegistry registry;`
- `private final LoaderRunHistoryService runHistory;`
- `private final DocumentIdHasher hasher;`
- `@Qualifier("loaderJdbcTemplate") private final JdbcTemplate loaderJdbc;`

Public method: `public void run(LoaderJobDefV4 def)`. Behavior:
1. `Instant startedAt = runHistory.recordRunStart(def.getKey())` — INSERTs the RUNNING row, returns the PK timestamp.
2. `long t0 = System.nanoTime()`.
3. `AtomicLong rowCount = new AtomicLong(0)`.
4. `BulkIngester<String> ingester = registry.ingesterFor(def.getKey())` — must be non-null (Plan 04 Task 1 invariant).
5. `LoaderBulkListener listener = registry.listenerFor(def.getKey())`.
6. Try block:
   - `loaderJdbc.query(con -> { PreparedStatement ps = con.prepareStatement(def.getSource().getQuery()); ps.setFetchSize(1000); return ps; }, (RowCallbackHandler) rs -> { Map<String, Object> row = rowToMap(rs); String docId = hasher.hash(def.getSource().getPrimaryKey(), row); ingester.add(op -> op.index(idx -> idx.index(def.getTarget().getAlias()).id(docId).document(row)), def.getKey() + ":" + docId); rowCount.incrementAndGet(); })` — exact pattern from research Pattern + Phase 5 SqlQueryServiceV4.
   - `ingester.flush()` — force drain of the in-flight queue before declaring success. NOT `close()` — close belongs to `@PreDestroy` on the registry (Pitfall L3).
   - `long durationMs = Duration.ofNanos(System.nanoTime() - t0).toMillis()`.
   - If `listener.getFailedItemCount() > 0`, treat as partial failure: call `runHistory.recordRunFailure(def.getKey(), startedAt, durationMs, "Bulk listener recorded " + count + " failed items — see app logs for details")` and log a warning. Otherwise `runHistory.recordRunSuccess(def.getKey(), startedAt, rowCount.get(), durationMs)`.
7. Catch `Throwable` (the broad catch is intentional — research Anti-Patterns says "silent failure is worse than no loader"; we want to record FAILED for any uncaught exception):
   - `long durationMs = Duration.ofNanos(System.nanoTime() - t0).toMillis()`.
   - Build error message: `t.getClass().getSimpleName() + ": " + t.getMessage()`. Append the first 4 stack frames via `Arrays.stream(t.getStackTrace()).limit(4).map(StackTraceElement::toString).collect(joining("\\n"))`.
   - `runHistory.recordRunFailure(def.getKey(), startedAt, durationMs, errorMessage)` — service truncates to 8 KB internally.
   - `log.error("Loader job [{}] failed", def.getKey(), t)` — full stack trace in app log; the truncated message in DB is for fast admin-endpoint surface.
   - Do NOT rethrow — the ticker's `executeWithLock` would also catch it; cleaner to consume here.

Private helper: `private Map<String, Object> rowToMap(ResultSet rs) throws SQLException`:
- Reads `ResultSetMetaData md = rs.getMetaData()`. For i in 1..md.getColumnCount(): `String col = md.getColumnLabel(i).toLowerCase()`, `Object val = rs.getObject(i)`. Build LinkedHashMap to preserve declaration order. Return the map.
- Lowercase keys match Phase 5 SqlQueryServiceV4 convention (Oracle returns uppercase; ES `_source` uses lowercase).

Javadoc cites: D-6.4 (deterministic ID via hasher), D-6.16 (primary DS), Pitfall L10 (streaming via RowCallbackHandler + setFetchSize avoids OOM), and the broad catch is justified by 06-RESEARCH.md Anti-Patterns.

Returns `TaskResult<Void>` so the controller can map `wasExecuted()` to HTTP 200 vs 409. Note: the registry's `markFired` is NOT called inside `runNow` — a manual trigger should not affect the next scheduled fire calculation. Document this contract clearly in the Javadoc.
