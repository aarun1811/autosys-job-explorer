Document ID hashing (Pitfall L5 — research is explicit):
- Hash inputs: `objectMapper.writeValueAsBytes(List.of(rowValue(pk1), rowValue(pk2), ...))` — JSON encoding of the PK array, ordering as declared in config. This eliminates the separator-collision class.
- Algorithm: `MessageDigest.getInstance("SHA-256")`, take first 8 bytes of digest, format as `String.format("%016x", longFromFirst8Bytes)` — 16 lowercase hex chars.
- Alternative formulation: `HexFormat.of().formatHex(digest, 0, 8)` (JDK 17+) — same result, fewer LOC.

Add a `@Bean JdbcTemplate loaderJdbcTemplate(@Qualifier("dataSource") DataSource ds)` inside LoaderRunHistoryService's own @Configuration nested class, OR (preferred) declare a top-level bean in `LoaderShedLockConfig` (Plan 04) and inject by name into LoaderRunHistoryService. Run-history Oracle SQL (from 06-RESEARCH.md Code Examples):
- INSERT: `INSERT INTO loader_run_history (job_key, started_at, status) VALUES (?, ?, 'RUNNING')`
- UPDATE on success: `UPDATE loader_run_history SET finished_at = ?, status = 'SUCCESS', row_count = ?, duration_ms = ? WHERE job_key = ? AND started_at = ?`
- UPDATE on failure: same shape with status='FAILED' + `last_error` set.
- Prune (Oracle 19c+ analytic):
  ```sql
  DELETE FROM loader_run_history
  WHERE job_key = ?
    AND (job_key, started_at) NOT IN (
      SELECT job_key, started_at FROM (
        SELECT job_key, started_at,
               ROW_NUMBER() OVER (PARTITION BY job_key ORDER BY started_at DESC) AS rn
        FROM loader_run_history
        WHERE job_key = ?
      )
      WHERE rn <= 20
    )
  ```
- Fetch last N: `SELECT * FROM (SELECT * FROM loader_run_history WHERE job_key = ? ORDER BY started_at DESC) WHERE ROWNUM <= ?`.
- `last_error` truncation: cap to 8192 chars before bind (research A8).
