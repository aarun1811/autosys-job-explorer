<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| External link → user browser (https://lnkd.in/...) | AppID anchor opens external URL in new tab |
| Backend → React (execution-order JSON) | Dialog renders backend response as `<pre>{JSON.stringify(data, null, 2)}</pre>` |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03.3-01 | Tampering / XSS | Dialog `<pre>` JSON rendering | mitigate | JSON.stringify with `<pre>{...}</pre>` (text content, NOT dangerouslySetInnerHTML) — React escapes by default. Untrusted backend strings cannot inject HTML. |
| T-03.3-02 | XSS | href / mailto attributes | mitigate | href is a static literal (`https://lnkd.in/...`) for AppID; mailto href is constructed via template literal but anchor `href="mailto:${value}"` does not execute scripts. `target="_blank" rel="noopener noreferrer"` prevents reverse-tabnabbing on AppID. |
| T-03.3-03 | XSS | title attribute interpolation | mitigate | title attribute is text content — React escapes; no script execution surface. |
| T-03.3-04 | Information disclosure | execution-order error path | mitigate | reportRequestFailure(err) renders only the 32-hex correlation ID in Sonner (per Phase 2 D-2.11). No stack traces or PII leaked. |
| T-03.3-05 | Tampering | encodeURIComponent on jobName | mitigate | Path segment `${encodeURIComponent(jobName)}` prevents path traversal / injection in the URL. |
</threat_model>
