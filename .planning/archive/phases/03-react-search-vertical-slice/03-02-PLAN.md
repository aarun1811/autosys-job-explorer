<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| URL → React | URL params are user-controlled (paste, manual edit). Zod validation lives on the route (Plan 07); this plan reads validated values. |
| localStorage → React | LocalStorage is per-origin and user-controllable; this plan reads/writes only string arrays under one key. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03.2-01 | Tampering | localStorage recent-searches | mitigate | read() helper JSON.parse in try/catch; non-array → []; non-string entries filtered out via `filter(s => typeof s === 'string')`. Malformed payload yields empty list, never a crash. |
| T-03.2-02 | Denial of service | localStorage quota | mitigate | setItem in try/catch (swallow); in-memory state updates regardless. Hook continues to function even if storage is full. |
| T-03.2-03 | Information disclosure | recent-searches contents | accept | Per-origin localStorage; same-origin policy isolates. Internal-tool context; no PII concern in search terms (file names, job names). |
| T-03.2-04 | Tampering | URL q/cat params | mitigate | Zod `validateSearch` on the route (Plan 07) parses/sanitizes; useSearchState reads only typed output. |
</threat_model>
