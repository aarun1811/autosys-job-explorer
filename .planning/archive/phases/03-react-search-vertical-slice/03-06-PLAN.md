<objective>
Ship the search chrome: `SearchBar` (input + clear + recent-searches Popover), `SearchToolbar` (result-count Badge + Excel export DropdownMenu), and `CategoryTabBar` (single non-interactive tab — Phase 4+ seed). These three components compose the layout above the grid; SearchPage (Plan 07) glues them to SearchGrid (Plan 05).

Purpose:
- SEARCH-03 (URL-synced state): SearchBar consumes `useSearchState` to round-trip `q` through the URL — but state ownership is delegated to SearchPage (which holds the submit handler). SearchBar is a controlled input with an `onSubmit` prop.
- SEARCH-04 (Excel export parity): SearchToolbar's DropdownMenu fires `gridApi.exportDataAsExcel()` (client-side, AG-Grid Enterprise built-in — D-3.10 acknowledges SSRM cached-rows-only limitation vs Angular's backend export).
- SEARCH-05 (recent searches): SearchBar renders a Popover anchored to the input, listing the 10 most-recent terms from `useRecentSearches`.

Output: 3 component files + 2 test files (CategoryTabBar is trivial — covered by SearchPage integration in Plan 07).
</objective>
